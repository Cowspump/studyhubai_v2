Этот вариант лучше всего подходит для генерации структуры всего приложения и ключевых компонентов.

Промпт:
"Действуй как Senior Fullstack Developer и Lead UI/UX Designer. Мне нужно спроектировать админ-панель для платформы StudyHubAI.

Цель: Создать интерфейс управления для суперадмина, который контролирует преподавателей и их финансовые показатели.

Основные функции:

User Management: Создание/редактирование аккаунтов преподавателей (RBAC).

Group & Student Tracking: Просмотр групп, созданных преподавателями, и списков их студентов.

Billing Dashboard: Детальная статистика по каждому преподавателю (кол-во студентов, активные часы, используемый функционал) для формирования счетов.

Технические требования:

Stack: React (v18+), TypeScript, Tailwind CSS.

Best Practices: Использование декомпозиции компонентов, кастомных хуков для логики, архитектуры типа Feature-Sliced Design (FSD).

UI/UX: Дизайн в стиле 'Modern SaaS' (минимализм, качественная типографика, много 'воздуха', мягкие тени). Визуально панель должна выглядеть дороже и технологичнее, чем текущая главная страница.

Components: Использование современных библиотек (например, Shadcn/UI или Radix UI) для интерактивных элементов.

Сгенерируй структуру папок проекта и пример кода для страницы 'Teacher Analytics' с использованием Recharts для визуализации статистики."
